## 2 演習2-1 : Fitts の法則
本演習では，Fittsの法則に関する実験を行う．なお，Fittsの法則についての説明は，別途講義で説明する．

### 2.1 実験用プログラムの確認
まず，実験で使うプログラムを確認する．

実験用プログラムのウインドウ内には以下が表示される．
*   **円周上に配置されている複数の円**: これが選択のターゲットとなる．以下，区別する際には，ターゲットとなる円を小円，小円が配置される円周のことを大円と呼ぶことにする．
*   **3つの設定項目**: 大円の直径 (距離，DISTANCE)，小円の直径 (幅，WIDTH)，小円の数 （# CIRCLES）が設定できる．設定を変更する際には，各項目の数値を変更してから SETボタンを押す．
*   **START ボタン**: ウィンドウ右下にあるこのボタンを押すことで，選択時間測定が開始される．

STARTボタンを押すと，実験が開始される．START を押した瞬間に，円の一つが緑色に変わるので，それをマウスで選択 (クリック)すると，別の円が緑に変わるので，次はそれをクリックする．これを続けていき，全ての円をクリックすると終了する．このとき，各クリック時刻間の時間がコンソールに表示される．また，結果が csvファイルに保存される．

実験の条件設定としては、距離 (大円の直径)の 3条件と幅 (小円の直径)の 3条件を組み合わせて，表 1のように 3×3の計 9つの条件を設定し，各条件での平均選択時間を実験で求める．

### 2.3 結果の取得
上記のように設定を変更しつつ，選択時間の計測実験を行うと，各設定毎に別々の csvファイルに実験結果が書き込まれる．ファイルのフォーマットは以下の通りである．

```csv
try,time,mouseMoveDistance,miss
1,579,132.46,0
2,501,134.63,0
3,548,130.68,0
4,558,122.67,0
...<以下省略>...
```

1行目はラベルであり各項目の意味は，(1)試行番号，(2)選択にかかった時間 (ミリ秒)，(3)実移動距離，(4)選択ミスか否か (ミスなら 1，そうでなければ 0)，である．
2行目以降の 3番目の項目はターゲットを選択する際の実移動距離である．実験における設定距離は大円の直径であり，実際にクリックしていく際の実移動距離ではない．そのため，本来は実移動距離を用いて分析を行うべきである．しかし，今回はひとまず簡単のため設定距離 (大円の直径)を移動距離として用いて分析するので良い．

### 2.4 分析
実験で得られたデータを使って，選択時間とターゲット間の距離・幅の関係を調査し，フィッツの法則が成り立っているかどうかを確かめよう．具体的には，以下の手順で分析する．

1.  **平均選択時間とIDの計算**: 各条件毎に平均選択時間とフィッツの法則における ID(Index of Difficulty)を計算する．各条件毎に複数回計測した場合には，計測毎のMTを求める．
    *   平均選択時間:

      $$
      MT = \frac{\sum_{i=1}^{N} t_i}{N}
      $$

    *   ID(Index of Difficulty): 距離 D, 幅 W とすると

      $$
      ID = \log_2\left(\frac{D}{W} + 1\right)
      $$
    
2.  **散布図の作成**: 横軸を ID, 縦軸を平均選択時間MTとして，実験結果をプロットしたグラフ (散布図)を作成する．各条件毎に複数のMTを求めた場合には，条件 (ID)毎に複数の点をプロットする．
3.  **近似直線の算出**: プロットされた点を線形近似する直線の式を求める．つまり，

    $$
    MT = a + b \times ID
    $$

    となる *a*, *b* を求める．また，求めた直線をグラフ上に描画してみよう．
4.  **スループット (TP)の算出**: スループット (Throughput, TP)を求める．スループットは実験参加者がタスクを行った際の能率であり，以下の式で求められる．

    $$
    TP = \frac{ID}{MT}
    $$

条件 (ID)・計測毎に TPを求めて，どう変化したかをグラフにしてみよう．

これらの計算やグラフ作成等はエクセルで行うことができる．(もちろん，Rや Pythonなどを用いてもよい．)

## 3 演習2-2 : 電卓
演習 2-2では，KLMおよび演習 2-1の結果を用いて，簡単なGUIプログラムにおける入力時間の予測を行う．また，それが実際にどの程度妥当であるかをユーザ実験を行い，確かめる．なお，KLMについては別途講義で説明する．


### 3.1 実験用プログラムの確認
まず，実験で使うプログラムを確認する．
本実験で用いるプログラムは電卓の簡易版である．テンキーを模したボタンを使って四則演算が行える．また，各キーのサイズとキー間の距離をスライダーで変更することができる．実験用の機能として，各キーを入力した際の時間 (直前のキーを押した時間から該当キーを押すまでの時間．単位ミリ秒)とキー入力時の移動距離 (直前のキーを押した位置から該当キーを押した位置までの距離)がログとして記録される．

### 3.2 入力タスクと入力時間の計測
実験のタスクは，提示される計算式を電卓GUI上で入力することである．複数回の入力タスクを行い，1タスク当たりの入力時間を平均することで，1タスクの平均入力時間を求める．なお，今回は提示された計算式 1つ (例：23+45)を入力することを 1タスクとする．1回の実験あたり8回計算式が提示される．

1.  実験開始前に，キーのサイズおよび間隔を決める．それぞれの値を上部のスライダーで調整すると，キーのサイズと距離が変更される．
2.  STARTボタンを押して，実験を開始する．
3.  すると，「．．．．．」と表示されていた場所（Text to be input）に，「45-86」といった計算式が表示される．
4.  表示された計算式をキー入力する．入力したキーの値はresult(計算機としての表示部)に表示される．なお，入力を間違っても修正しないで，そのまま入力を進める．なおテンキー内の「C」は1文字削除ではなく実験自体を取り消す（実験を中止し記録中のログを削除する）仕様となっているため注意すること．
5.  表示された計算式を入力完了したら，「=」キーを入力する．すると，次の計算式が表示されるので，続けて入力を行う．表示される式はランダムに生成される．
6.  計算式を 8回入力すると終了する．

startボタンを押してから，それ以降の各キーの入力時間 (ミリ秒)と移動距離は csv ファイルに記録される．実験終了後にstartボタン下のDownload CSVというボタンが押せるようになり，そこからファイルをダウンロードできる．
本実験のタスクは演習 2-1のタスクよりも複雑であるため，本番前に少し練習して慣れておくと良い．

### 3.3 実行時間の予測

**KLMによる予測**
本実験の 1タスク (1つの計算式の入力)にかかる時間をKLMを使って予測してみよう．最初に提示される計算式を読むのでその時間をMに対応させる．その後はカーソルをキーまで動かして，マウスボタンを押して離すという操作の繰り返しになる．

    $$
    M → P → 2×B → P → 2×B...
    $$

1タスク中でキーを押す回数がN であるとすると， 

    $$
    = M + N × (P + 2B)
    $$ 

となるだろう．そこで，この式を使って，今回の実験タスクにおける入力時間の予測を計算しよう．

**Fittsの法則の実験結果を適用した予測**
もう少し予測の精度を上げるために演習 2-1の結果を適用してみよう．演習 2-1の結果を用いれば，キー間の距離とサイズに応じてキーを押す時間が変化することを予測に反映させられるはずである．演習 2-1ではマウスボタンを押す時間は考慮していなかったが，KLM的に考えれば演習 2-1も P + 2B の時間を計測していることになる．従って，演習 2-1で求めた近似式を今回の予測に適用する際には P + 2Bの部分の時間を演習 2-1で求めた式で計算すれば良いだろう．
そこで，次のように入力時間を予測しよう．

1.  実際にどのキーをどの順番で押すかは，入力する計算式に依存するので，実験するまでは不明である．従って，マウスの実移動距離も事前には不明である．そこで，GUIにおけるキー間距離の平均を使って予測してみよう．今回のタスクは，START → 数字 → 数字 → 演算子 → 数字 → 数字 → =という順番でキーを押すことになるので，以下のようになる．
    *   START → 数字
    *   数字 → 数字
    *   数字 → 演算子
    *   演算子 → 数字
    *   数字 → =
    それぞれの場合について，各自の実験設定におけるキーの中心座標間距離の平均を計算する．例えば，START → 数字の場合，STARTボタンは 1箇所であるが，数字キーは 10箇所あるので，STARTボタンからそれぞれのキーまでの距離を計算して平均する．等確率で数字が出現するとすれば，多数回タスクを行った際の実移動距離の平均は，この計算で求めた平均値に近くなると期待できる．なお，各キーのサイズや中心座標等の情報は，UI下部の「Button Info」にて確認できる（デフォルトは折りたたまれているため開いて確認する）．また，実験実施後に書き出される csvファイル中にも同情報が出力される．
2.  1で求めたボタン間の距離の平均値と実験で設定したボタンのサイズを使って Fittsの法則における IDを計算すれば，演習 2-1で求めた近似式にあてはめることができる．タスク内の各キーを押す時間を計算して合計することで入力予測時間を計算してみよう．

**実際の移動距離から推定される入力時間の計算**
今回の実験においてランダムに生成されたタスクでの実際のボタン間距離は，上記で計算したボタン間距離の平均値とズレているかもしれない．そこで，実験結果の csvファイル内に記載されている実験時のマウスの実移動距離の値を用いて，同様に入力にかかる時間を推定してみよう．
